## 🚀 快速开始 (Quick Start)

### 方式 A：云端开发 (推荐 / Cloud Development)
无需安装任何本地环境，直接在浏览器中开始：
1. 点击本仓库右上角的绿色 **Code** 按钮。
2. 选择 **Codespaces** 选项卡，点击 **Create codespace on main**。
3. 等待容器初始化完成后，在终端直接输入 `hexo s` 即可通过转发端口预览。



### 方式 B：本地开发 (Local Development)
如果你偏好本地环境，请确保已安装 **Node.js (v18+)**、**Git** 和 **Hexo-cli**。

1. **克隆仓库**:
   `git clone https://github.com/oxarlly/hexo-theme-butterfly-usk.git`
2. **安装依赖**:
   `cd hexo-theme-butterfly-usk && npm install`
3. **启动预览**:
   `hexo s`

## 🛠️ 脚本自动化 (Automation)

本项目内置了针对 Windows 用户优化的自动化脚本：

- **运行脚本.bat**: 集合了环境安装、清理缓存、本地测试、一键部署四大核心功能。
- **创建文章或页面.bat**: 引导式交互创建新的 .md 博文或独立页面，无需记忆繁琐的 CLI 命令。

---

## 🌐 部署指南 (Deployment)

1. **修改站点配置**: 打开根目录 `_config.yml`，定位至 `url` 字段，修改为你的实际域名。
2. **配置 Git 仓库**: 找到 `deploy` 字段，将 `repo` 地址修改为你自己的 GitHub 仓库 URL。
3. **执行发布**: 
   在终端运行 `hexo cl && hexo g && hexo d` 或通过脚本一键上传。



---

## 🎨 配置说明 (Configuration)

本项目采用 **配置分离 (Config Overriding)** 模式，确保主题升级不丢失数据：

- **基础配置 & 看板娘**: 位于根目录 `_config.yml`。
- **主题功能设置**: 位于根目录 `_config.butterfly.yml`。
- **更多进阶配置**: 请参考 [Butterfly 官方配置指南](https://butterfly.js.org/posts/4aa8abbe/)。
