01. 云端基础设施 CLOUD INFRASTRUCTURE

GitHub Codespace 提供了一个基于容器 Container 的完整开发环境。通过这种方式，我们可以摆脱本地环境的不一致性。

[插图：GitHub Codespace 架构与容器运行原理图]

开启步骤：进入 GitHub 仓库，选择 Codespaces 并点击 Create codespace on main。

02. 环境搭建 ENVIRONMENT SETUP

在 Codespace 终端执行以下指令。注意代码块中的命令已支持超长自动换行。

1
2
3
4
5
npm install -g hexo-cli hexo init . npm install # 安装 Butterfly 必要插件 npm install hexo-renderer-pug hexo-renderer-stylus --save
重点提示： 必须安装渲染器 Renderer，否则 Butterfly 主题将无法正常显示，页面会报 Pug 错误。

03. 主题定制 CUSTOMIZATION

拉取主题源码并将其关联至你的 Hexo 项目：

1
git clone -b master https://github.com/JerryC808/hexo-theme-butterfly themes/butterfly

在配置文件 _config.yml 中，找到 theme 属性并修改为 butterfly。

04. 配置文件详解 CONFIGURATION

Butterfly 官网推荐使用配置文件覆盖机制 Config Overriding。在根目录下创建 _config.butterfly.yml。

1
2
3
4
5
6
7
8
9
# 网站图标 (Favicon) favicon: /img/favicon.png # 顶部导航栏配置 (Nav Menu) menu: 首页: / || fas fa-home 文章: /archives/ || fas fa-archive 标签: /tags/ || fas fa-tags 分类: /categories/ || fas fa-folder-open

05. 自动化部署 DEPLOYMENT

通过 hexo-deployer-git 插件，我们可以将生成的静态网页推送到 GitHub Pages 托管服务。

1
2
3
4
5
6
deploy: type: git repo: https://github.com/YourName/YourName.github.io.git branch: main message: "Site updated by Codespace"
最后一步： 在终端运行 hexo cl && hexo g && hexo d 即可完成一键发布。
